# Ruins POI — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add ruins as the second POI type — large ancient structures with decay, overgrowth, edge connections for multi-tile ruins, and wall tinting.

**Architecture:** RuinPlanner selects 1-2 large structure types and positions them. BuildingGenerator (reused, high decay) places them. RuinDecay post-processes: extra wall collapse, rubble, overgrowth, wall tinting. EdgeConnector builds wall segments at connected tile edges with passage gaps. PathRouter links structures. All wired through poi_phase's new OW_Ruins branch.

**Tech Stack:** C++20, `cmake -B build -DDEV=ON && cmake --build build`

**Spec:** `docs/superpowers/specs/2026-04-08-ruins-poi-design.md`

### Task 2: RuinPlanner

Plans which structures to place and where.

**Files:**
- Create: `include/astra/ruin_planner.h`

- Modify: `CMakeLists.txt`

- [ ] **Step 1: Create header**

```cpp
#pragma once

#include "astra/settlement_types.h"
#include "astra/terrain_channels.h"

#include <random>

namespace astra {

struct RuinPlan {
    PlacementResult placement;
    CivStyle style;
    std::vector<BuildingSpec> structures;
    std::vector<PathSpec> corridors;       // between structures
    int connected_edges = 0;              // bitmask: N=1, S=2, E=4, W=8
};

class RuinPlanner {
public:
    RuinPlan plan(const PlacementResult& placement,
                  const TerrainChannels& channels,
                  const TileMap& map,
                  const MapProperties& props,
                  std::mt19937& rng) const;
};

} // namespace astra
```

- [ ] **Step 2: Implement RuinPlanner**

The planner:
1. Selects CivStyle: `civ_ruin_default()` normally, `civ_ruin_heavy()` when `lore_plague_origin` or `lore_scar_count > 0`
2. Determines connected edges from `detail_neighbor_n/s/e/w == Tile::OW_Ruins`
3. Picks 1-2 structure types randomly from {Temple, Vault, GreatHall, Archive, Observatory}
4. Generates BuildingShape for each using size ranges:
   - Temple: 18-24 × 10-14
   - Vault: 12-16 × 8-12
   - GreatHall: 20-28 × 12-16
   - Archive: 14-18 × 8-10
   - Observatory: 10-14 × 10-14
5. Positions structures in the core zone (center of footprint), separated by 4+ tiles
6. Plans corridors between structures (PathSpec, width 3)
7. Stores connected_edges bitmask for EdgeConnector

Shape generation reuses the same `make_shape` pattern from SettlementPlanner but with larger sizes and the door facing the nearest other structure.

### Task 3: DecayContext + RuinDecay

Post-processing pass that destroys and overgrows.

**Files:**
- Create: `include/astra/ruin_decay.h`
- Create: `src/generators/ruin_decay.cpp`
- Modify: `CMakeLists.txt`

- [ ] **Step 1: Create header**

```cpp
#pragma once

#include "astra/tilemap.h"
#include "astra/settlement_types.h"

#include <random>

namespace astra {

struct DecayContext {
    float age_decay = 0.5f;       // wall removal probability (beyond CivStyle decay)
    bool battle_scarred = false;  // future
    int blast_direction = -1;     // future
    bool seismic = false;         // future
};

class RuinDecay {
public:
    // Run decay on all structures in the footprint area.
    // Collapses walls, places rubble, adds overgrowth, sets wall tinting flags.
    void apply(TileMap& map, const Rect& footprint,
               const DecayContext& ctx, Biome biome,
               std::mt19937& rng) const;
};

} // namespace astra
```

- [ ] **Step 2: Implement RuinDecay**

The `apply()` function iterates the footprint area and:

1. **Extra wall collapse**: for each Wall/StructuralWall tile, roll `age_decay` probability. If hit, replace with Floor tile and place a Debris fixture (rubble).

2. **Rubble placement**: where walls were removed, place Debris. Also randomly place Debris on ~5% of empty floor tiles near walls.

3. **Overgrowth**: place flora fixtures on indoor floor tiles based on biome. Use the density table from the spec:
   - Jungle: 20% → FloraGrass, FloraHerb, FloraFlower
   - Forest: 12% → FloraGrass, FloraMushroom, FloraHerb
   - Grassland: 10% → FloraGrass, FloraFlower
   - Marsh: 15% → FloraGrass, FloraHerb
   - Rocky: 3% → FloraLichen
   - Volcanic: 0%
   - Ice: 2% → FloraLichen
   - Sandy: 3% → FloraGrass
   Flora placement uses `add_fixture` which sets Tile::Fixture — that's fine for interior tiles.

4. **Wall tinting**: for each surviving wall tile in the footprint, set bit 1 of `custom_flags_` via `set_custom_detail(x, y, ...)`. Since `custom_flags_` is a uint8_t and currently only uses bit 0, we need a new setter that can set arbitrary bits. Add `set_custom_flag(x, y, uint8_t bit)` to TileMap.

- [ ] **Step 3: Add `set_custom_flag` / `has_custom_flag` to TileMap**

In `include/astra/tilemap.h`, add to the public interface:

```cpp
    void set_custom_flag(int x, int y, uint8_t bit);
    bool has_custom_flag(int x, int y, uint8_t bit) const;
```

Implement in `src/tilemap.cpp`:

```cpp
void TileMap::set_custom_flag(int x, int y, uint8_t bit) {
    if (x < 0 || x >= width_ || y < 0 || y >= height_) return;
    if (custom_flags_.empty()) custom_flags_.resize(width_ * height_, 0);
    custom_flags_[y * width_ + x] |= bit;
}

bool TileMap::has_custom_flag(int x, int y, uint8_t bit) const {
    if (x < 0 || x >= width_ || y < 0 || y >= height_) return false;
    if (custom_flags_.empty()) return false;
    return (custom_flags_[y * width_ + x] & bit) != 0;
}
```

Define the flag constant: `static constexpr uint8_t CF_RUIN_TINT = 0x02;` in `tilemap.h` or `ruin_decay.h`.

### Task 4: EdgeConnector

Extends structures to connected tile edges.

**Files:**
- Create: `include/astra/edge_connector.h`
- Create: `src/generators/edge_connector.cpp`
- Modify: `CMakeLists.txt`

- [ ] **Step 1: Create header**

```cpp
#pragma once

#include "astra/settlement_types.h"
#include "astra/ruin_planner.h"

#include <random>

namespace astra {

class EdgeConnector {
public:
    // Build edge walls with passage gaps on connected edges,
    // and carve corridors from gaps to the nearest structure.
    void connect(TileMap& map, const RuinPlan& plan,
                 const MapProperties& props,
                 std::mt19937& rng) const;
};

} // namespace astra
```

- [ ] **Step 2: Implement EdgeConnector**

For each connected edge (checked via `plan.connected_edges` bitmask):

1. **Edge wall**: place Wall tiles along the map boundary. For N/S edges: horizontal wall at y=0 or y=h-1. For E/W: vertical wall at x=0 or x=w-1. Coverage ~60% with organic noise offset (use `fbm` from noise.h, same pattern as PerimeterBuilder).

2. **Passage gaps**: 2-3 gaps per edge, each 4-5 tiles wide. Gap positions seeded deterministically:
   ```cpp
   unsigned gap_seed = static_cast<unsigned>(props.overworld_x) * 7919u
                     + static_cast<unsigned>(props.overworld_y) * 104729u
                     + direction * 31u;
   ```
   For the shared edge between two tiles, both tiles compute the same seed. When tile C checks its west edge, it uses tile A's overworld_x (which is `props.overworld_x - 1` for the west neighbor). The key insight: for the edge between A(east) and C(west), both use `min(A.x, C.x)` as the seed base, plus a consistent direction constant.
   
   Simpler: use `min(own_x, neighbor_x)` for E/W edges and `min(own_y, neighbor_y)` for N/S edges as the seed. This ensures both tiles sharing an edge get the same seed.

3. **Corridors inward**: from each gap center, carve a winding 3-wide path (using the same noise-based winding as PathRouter's wide paths) toward the nearest structure in `plan.structures`. Path target is the closest door position of any structure.

### Task 5: Wall Tinting in Renderer

Make the renderer apply moss/crumble colors on ruin walls.

**Files:**
- Modify: `src/map_renderer.cpp` — pass custom flag to render descriptor
- Modify: `src/terminal_theme.cpp` — apply tinting when flag is set

- [ ] **Step 1: Pass ruin flag to renderer**

In `src/map_renderer.cpp`, find where Wall tiles build their RenderDescriptor. The seed is already computed from `position_seed(mx, my)`. Check the `has_custom_flag(mx, my, CF_RUIN_TINT)` and encode it into the descriptor's flags. Use an unused RenderFlag bit or encode in the seed.

Simplest approach: add a new `RF_RuinTint` flag to `RenderFlag` in `include/astra/render_descriptor.h`. But all 8 bits are used. Instead, encode it in the seed's top bit — the seed has 8 bits and position_seed uses all of them for visual variation. We can sacrifice 1 bit: use bit 7 as the ruin tint flag, leaving 7 bits (128 values) for variation.

In `map_renderer.cpp`, when building the descriptor for a Wall tile:
```cpp
uint8_t s = position_seed(mx, my);
if (map.has_custom_flag(mx, my, CF_RUIN_TINT)) {
    s |= 0x80;  // set top bit
}
desc.seed = s;
```

- [ ] **Step 2: Apply tinting in terminal_theme.cpp**

In the wall rendering section (around line 1231 where `Tile::Wall` is handled), check the top bit:

```cpp
if (tile == Tile::Wall) {
    const char* utf8 = wall_glyph_for_biome(biome, seed & 0x7F);
    bool ruin_tint = (seed & 0x80) != 0;
    Color c;
    if (ruin_tint) {
        int tint = (seed & 0x7F) % 3;
        switch (tint) {
            case 0: c = bc.wall; break;                             // normal
            case 1: c = static_cast<Color>(28); break;              // mossy green
            case 2: c = static_cast<Color>(94); break;              // crumbling brown
        }
    } else {
        c = bc.wall;
    }
    if (remembered) c = bc.remembered;
    return {'#', utf8, c, Color::Default};
}
```

### Task 6: Wire Ruins into POI Phase

Connect everything through poi_phase.cpp.

**Files:**
- Modify: `src/generators/poi_phase.cpp`
- Modify: `include/astra/poi_phase.h` (if needed)

- [ ] **Step 1: Add OW_Ruins branch**

In `poi_phase()` in `poi_phase.cpp`, add a branch for ruins after the settlement branch. The ruin branch:

1. Check `props.detail_poi_type == Tile::OW_Ruins`
2. Determine footprint: ruins use a larger footprint than settlements (120×70 default)
3. Call `PlacementScorer::score()`
4. Call `RuinPlanner::plan()`
5. Apply terrain mods (Level + Clear for the core zone)
6. Call `BuildingGenerator::generate()` for each structure with ruin CivStyle
7. Call `PathRouter::route()` for corridors between structures (using plan.corridors converted to SettlementPlan.paths)
8. Call `RuinDecay::apply()` — post-process the whole footprint
9. Call `EdgeConnector::connect()` — build edge walls and corridors
10. Call `ExteriorDecorator::decorate()` — minimal: just scatter clearing, no lamps/benches
11. Return footprint rect (same as settlement)

Note: RuinDecay runs AFTER buildings are placed so it can destroy their walls. EdgeConnector runs AFTER decay so edge walls are intact (they're new construction, not ancient).

The ruin pipeline needs to construct a `SettlementPlan`-compatible structure to reuse PathRouter and ExteriorDecorator. Simplest: build a `SettlementPlan` from the `RuinPlan` — set style, buildings, paths, no perimeter.
