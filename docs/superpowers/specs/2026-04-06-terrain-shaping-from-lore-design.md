# Terrain Shaping from Lore — Design Spec

**Date:** 2026-04-06
**Status:** Draft
**Roadmap item:** Terrain shaping from lore — megastructures as orbital POIs, beacons as unique landmarks, terraforming alters biome, weapon tests scar terrain

## Overview

Lore events generated by the galaxy simulation (terraforming, wars, weapon tests, beacon/megastructure placement) should visibly reshape planetary surfaces at every zoom level: overworld terrain tiles, detail map scatter/structures, and star chart markers. The system uses **influence maps** as the data bridge — lore produces 2D float fields, terrain generators consume them.

### Design Decisions

- **Full layered shaping** — lore affects terrain tiles, POIs, AND detail maps (not just POI counts)
- **Hybrid biome blending** — terraformed worlds show alien biome patches over the natural base terrain, not full replacement. Coverage degrades with age.
- **Severity-scaled scarring** — scar count determines devastation coverage. One weapon test = one crater zone. Prolonged war = overlapping devastation.
- **Per-system flags with scar count** — no per-body scar propagation. The sim works at system level; the overworld generator rolls scar zone placement.
- **Both orbital + ground for landmarks** — megastructures/beacons get star chart markers (cosmetic) AND surface POI stamps (gameplay content).
- **Influence map architecture** — lore pre-computes float arrays, terrain generators consume them. Clean separation of concerns, order-independent, visualizable in dev mode.
- **Deterministic shared-edge seeding** — detail map zones use shared edge seeds for cross-zone terrain connectivity.

### Bundled Fixes

Two existing detail map bugs are fixed as part of this work:

1. **POI duplication across 9 zones** — POI structures only generate in center zone (1,1), not all 9.
2. **Zone-to-zone connectivity** — deterministic shared edge strips ensure rivers and terrain connect across zone boundaries.

### New fields in `LoreSystemData`

```cpp
                             //   1. megastructure_builder (if has_megastructure)
                             //   2. terraformed_by (if terraformed)

                             //   4. -1 (no lore presence)
```

### New fields in `LoreAnnotation`

```cpp

int primary_civ_index = -1;

```

### New fields in `MapProperties`

```cpp
int lore_scar_count = 0;
Architecture lore_civ_architecture = Architecture::Geometric;
int lore_primary_civ_index = -1;
float lore_alien_strength = 0.0f;   // detail map: alien influence at this cell

```

### Pipeline flow

```
GalaxySim::build_lore()
  -> LoreSystemData { scar_count, primary_civ_index, existing flags... }

apply_lore_to_galaxy()
  -> LoreAnnotation { scar_count, primary_civ_index, primary_civ_architecture, existing flags... }

Player lands on body (game_world.cpp):
  -> MapProperties { lore_scar_count, lore_civ_architecture, existing flags... }

LoreInfluenceMap generated from MapProperties before overworld generation

Player enters detail zone (game_world.cpp):
  -> MapProperties { lore_alien_strength, lore_scar_intensity } read from influence map at overworld cell
```

The five `Architecture` types (Crystalline, Organic, Geometric, VoidCarved, LightWoven) serve as the aesthetic key for all terrain shaping — determining alien biome palettes, scar visual styles, and landmark appearances.

## 3. Lore Influence Map

### Structure

A new struct `LoreInfluenceMap` generated once per landing, before the overworld generator runs. Same dimensions as the overworld grid.

| Channel | Type | Range | Meaning |
|---------|------|-------|---------|
| `alien_strength` | `float[]` | 0.0-1.0 | How much alien biome overrides natural terrain |
| `scar_intensity` | `float[]` | 0.0-1.0 | Devastation level (0 = untouched, 1 = ground zero) |
| `landmark_type` | `enum[]` | None/Beacon/Megastructure | Reserved zone for a major landmark POI |

Three flat arrays (no per-cell struct overhead). Passed to the overworld generator via `MapProperties` or as a companion object.

### Generation Logic

**Alien biome patches** (`lore_terraformed == true`):
- Seed 2-5 origin points randomly across the map
- Each origin gets a radial fBm noise field with irregular falloff
- Total coverage targets 30-60% of passable tiles
- Strength falls off from origin centers, creating ragged organic edges
- Older civilizations (earlier `epoch_end_bya`) get lower peak strength — their terraforming has degraded over billions of years

**Scar zones** (`scar_count > 0`):
- Place `scar_count` impact epicenters at random passable locations (avoiding landmarks)
- Each epicenter gets a radial falloff; radius scales with total scar count (1 scar = small crater, 5+ scars = overlapping devastation)
- Intensity is additive where zones overlap, clamped to 1.0
- A heavily bombarded world has zones of total devastation bleeding into each other

**Landmarks** (`lore_beacon` or `lore_megastructure`):
- Reserve a rectangular zone (~7x7 tiles) at a suitable location
- Elevated terrain for beacons, flat open ground for megastructure anchors
- `landmark_type` channel marks these cells so POI placement avoids overlap

## 4. Overworld Generator Consumption

Lore integrates into existing overworld generation phases — no new phases added.

### `generate_layout()` — Terrain classification

1. Classify normally (elevation + moisture) -> `natural_tile`
2. Read `alien_strength` at this cell
   - If `alien_strength > 0.15` (threshold to avoid speckling):
     - Look up alien biome palette for the civ's `Architecture` type
     - If `alien_strength > 0.6`: fully replace with alien tile
     - Between 0.15-0.6: probabilistic (`rng_float() < alien_strength` determines winner)
     - Creates organic ragged boundaries between alien and natural terrain
3. Read `scar_intensity` at this cell
   - If `scar_intensity > 0.1`:
     - Above 0.7: **glassed/crater** tile (impassable crater core or scorched glass)
     - 0.4-0.7: **scorched earth** tile (passable but barren)
     - 0.1-0.4: **scarred ground** (degraded version of natural tile)
   - Scar tiles override alien tiles (destruction post-dates terraforming)

### `place_features()` — POI placement

1. **Landmarks first** — read `landmark_type` channel, place beacon/megastructure stamp at marked zone
2. **Existing POI logic unchanged** — ruin count scaling by `lore_tier`, crashed ships on battle sites, etc.
3. **New exclusion** — POI placement skips cells where `landmark_type != None`
4. **Scar-zone POIs** — high-intensity scar zones (>0.7) get unique POI candidates: weapon debris, crash fragments, irradiated caves

### Rivers

Rivers generated during `generate_layout()` get interrupted by high-intensity scar zones (>0.7). `ensure_connectivity()` runs last as usual. Full river rework is deferred to a separate task.

## 5. Detail Map Generation

### Lore-modified biome scatter

When the overworld cell has `alien_strength > 0.15`, the detail map's `scatter_biome_features()` uses an alien scatter palette instead of the natural one. The `Architecture` type selects the palette:

| Architecture | Scatter style |
|---|---|
| Crystalline | Crystal formations, prismatic shards, resonating spires |
| Organic | Fleshy growths, pulsing nodes, tendril clusters |
| Geometric | Perfect geometric blocks, grid-etched slabs, aligned pillars |
| VoidCarved | Absence pits, floating debris markers, dark obelisks |
| LightWoven | Luminous threads, light pillars, glowing pools |

If `alien_strength` is in partial range (0.15-0.6), the detail generator mixes natural and alien scatter entries proportionally.

### Scar zone detail maps

When the overworld cell is scarred (`scar_intensity > 0.1`):
- `generate_layout()`: Higher wall density from rubble/debris. Crater-like terrain shaping with scorched aesthetic.
- `scatter_biome_features()`: Replaces natural scatter with devastation scatter — charred stumps, melted rock, weapon fragments, irradiated pools.
- Severity scales with `scar_intensity` value from `MapProperties`.

### Landmark detail maps

New POI type handlers in `place_features()`:
- **Beacon**: Central spire complex — wall ring tower surrounded by etched floor patterns and satellite pylons. Architecture-themed decoration. Portal/interaction point at center.
- **Megastructure anchor**: Massive foundation structure — thick walls forming partially buried geometric base with exposed machinery fixtures, access tunnels. Larger footprint than beacon.

### New `MapProperties` fields for detail maps

```cpp
float lore_alien_strength = 0.0f;   // from influence map at overworld cell
Architecture lore_alien_architecture; // civ aesthetic for alien scatter
float lore_scar_intensity = 0.0f;   // from influence map at overworld cell
```

Populated in `build_detail_props()` by reading the influence map at the corresponding overworld cell.

## 6. Detail Map Fixes

### Fix 1: POI duplication across 9 zones

**Problem:** `build_detail_props()` sets `detail_has_poi = true` for all zone coordinates when the overworld tile is a POI type.

**Fix:** Only set `detail_has_poi = true` when `zone_x == 1 && zone_y == 1` (center zone). The surrounding 8 zones generate as natural terrain of the same biome without POI structures.

### Fix 2: Zone-to-zone connectivity via shared edge seeding

**Problem:** Each of the 9 detail map zones is generated independently. Rivers, terrain, and features don't connect across zone boundaries.

**Fix: Deterministic shared edge strips.** Each edge between two adjacent zones is seeded from the shared edge coordinates, not the individual zone. Both zones generate the same edge strip independently and blend toward it from their respective sides.

Edge seed derivation:
```
edge_seed = overworld_seed ^ (ow_x * P1) ^ (ow_y * P2) ^ (edge_coord * P3)
```

Where `edge_coord` uniquely identifies which edge (e.g., vertical edge between zone columns 0 and 1). This is order-independent — doesn't matter which zone generates first.

The approach extends naturally across overworld tile boundaries: the edge between zone (2,1) of tile A and zone (0,1) of tile B derives its seed from both overworld coordinates.

Implementation integrates with the existing `EdgeWeights` / `compute_edge_weights()` system, which already blends terrain toward edges. The shared edge strip provides a concrete target to blend toward instead of just density interpolation.

A new header `include/astra/civ_aesthetics.h` centralizes all aesthetic lookups. All consumers reference this single source.

### Alien biome palettes

| Architecture | Alien Biome | Tile colors | Character |
|---|---|---|---|
| Crystalline | `AlienCrystalline` | Cyan, White, Bright Cyan | Sharp angular |
| Organic | `AlienOrganic` | Red, Magenta, Dark Red | Curved biological |
| Geometric | `AlienGeometric` | Yellow, White, Dark Yellow | Perfect geometric |
| VoidCarved | `AlienVoid` | Dark Magenta, DarkGray | Absence/void |
| LightWoven | `AlienLight` | Bright Yellow, White | Luminous glow |

### Scar biomes

| Biome | Colors | Character |
|---|---|---|
| `ScarredGlassed` | Dark Yellow, Orange, DarkGray | Melted glass, fused ground |
| `ScarredScorched` | DarkGray, Dark Red, Orange | Charred debris, ash |


Floor/Wall/Water tiles remain unchanged — the renderer picks different colors based on biome. The detail map uses these biomes for scatter palette selection, extending the existing `scatter_biome_features()` switch.

| `AlienLight` | `✦` `✧` `∗` `☆` | Bright Yellow, White, Bright White |

### Scar scatter glyphs

| Biome | Glyphs | Colors |
|---|---|---|
| `ScarredGlassed` | `▓` `░` `≈` `·` | Dark Yellow, Orange (208), DarkGray |
| `ScarredScorched` | `≠` `,` `~` `%` | DarkGray, Dark Red, Orange |

### Animations

New animation types added to `AnimationType` enum:

- **AlienPulse** — for Organic and LightWoven fixtures: slow color cycling (4-6 frame cycle, ~500ms/frame). Red-dark red-red for organic, yellow-white-yellow for light.
- **ScarSmolder** — for high-intensity scar zone fixtures: occasional flicker between base glyph and dim variant (3 frame cycle, ~800ms/frame). Applied to ~1 in 5 scar fixtures.
- **BeaconGlow** — beacon spire: pulsing bright glow cycling through civ palette colors. Visible as a color landmark from several tiles.
- **MegastructureShift** — megastructure anchor: slow alternation between two structural glyphs, suggesting ancient machinery still operating.

### Star chart orbital markers

Systems with megastructures or beacons get an icon in the local system view:
- `◈` for megastructure (colored by civ architecture)
- `⌾` for beacon (colored by civ architecture)

Cosmetic-only for now; can later become dockable locations.

## 11. Research Notes: Rivers V2 (Deferred)

Current river generation in `overworld_generator.cpp` has issues that should be addressed in a dedicated rework:

- **Unnatural flow:** Rivers are generated by finding mountain-adjacent high-elevation cells and flowing downhill, but the algorithm doesn't produce convincing drainage patterns. Rivers don't follow realistic watershed logic.
- **Detail map continuity:** Rivers at the detail map level use a sinusoidal water band (`OW_River` case in `detail_map_generator.cpp`, line 230-237). This doesn't connect across zone boundaries — a river can enter from the north edge of one zone and not appear at the south edge of the adjacent zone.
- **Scar interaction:** This spec calls for scar zones to interrupt rivers (high-intensity zones > 0.7 break them). The current implementation should handle this simply by overriding water tiles with scar tiles during `generate_layout()`. However, a proper Rivers V2 should model crater lakes (pooling on the upstream side of an interruption) and dried riverbeds (downstream side).
- **Terraform interaction:** Alien biome patches may overlay river tiles. Rivers V2 should decide whether alien terraforming changes water appearance (e.g., bio-pools for Organic civs, luminous streams for LightWoven) or leaves water untouched.
- **Shared edge seeding (from this spec):** The zone connectivity fix we're implementing here provides the infrastructure Rivers V2 needs. Once shared edge strips define boundary terrain, rivers can be made to connect by including water/river state in the edge strip data.

These notes should be referenced when planning the Rivers V2 task.
